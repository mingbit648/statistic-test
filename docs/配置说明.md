# 配置说明文档

## 概述

病历分析系统现在支持通过配置文件管理所有关键配置项，包括服务器设置、Dify API配置、文件上传设置等。

## 配置文件结构

### 主配置文件：`config/config.json`

```json
{
    "server": {
        "host": "0.0.0.0",
        "port": 8000,
        "debug": false
    },
    "dify": {
        "api_key": "your-dify-api-key-here",
        "base_url": "https://api.dify.ai/v1",
        "app_id": "your-app-id",
        "workflow_id": "your-workflow-id"
    },
    "file": {
        "upload_dir": "uploads",
        "max_file_size": 10485760,
        "allowed_extensions": [".xml", ".md", ".txt", ".json"]
    },
    "database": {
        "history_file": "data/history.json"
    }
}
```

### 配置项说明

#### 服务器配置 (server)
- `host`: 服务器监听地址，默认 `0.0.0.0`（所有网络接口）
- `port`: 服务器端口号，默认 `8000`
- `debug`: 调试模式，默认 `false`

#### Dify配置 (dify)
- `api_key`: Dify API密钥，必填项
- `base_url`: Dify API基础URL，默认 `https://api.dify.ai/v1`
- `app_id`: Dify应用ID
- `workflow_id`: Dify工作流ID

#### 文件配置 (file)
- `upload_dir`: 文件上传目录，默认 `uploads`
- `max_file_size`: 最大文件大小（字节），默认 10MB
- `allowed_extensions`: 允许的文件扩展名列表

#### 数据库配置 (database)
- `history_file`: 历史记录文件路径，默认 `data/history.json`

## 环境变量支持

系统支持通过环境变量覆盖配置文件中的设置：

### 服务器相关环境变量
- `SERVER_HOST`: 服务器主机地址
- `SERVER_PORT`: 服务器端口号
- `SERVER_DEBUG`: 调试模式（true/false）

### Dify相关环境变量
- `DIFY_API_KEY`: Dify API密钥
- `DIFY_BASE_URL`: Dify基础URL
- `DIFY_APP_ID`: Dify应用ID
- `DIFY_WORKFLOW_ID`: Dify工作流ID

## 使用方法

### 1. 修改配置文件

编辑 `config/config.json` 文件，设置您的实际配置：

```json
{
    "server": {
        "host": "0.0.0.0",
        "port": 8080,
        "debug": false
    },
    "dify": {
        "api_key": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
        "base_url": "https://api.dify.ai/v1",
        "app_id": "app-xxxxxxxxxxxxxxxx",
        "workflow_id": "workflow-xxxxxxxxxxxxxxxx"
    }
}
```

### 2. 使用环境变量（推荐用于生产环境）

```bash
# Windows PowerShell
$env:DIFY_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
$env:SERVER_PORT=8080

# 然后启动应用
python main.py
```

### 3. 配置优先级

环境变量 > 配置文件 > 默认配置

## 配置类API

### 获取配置值

```python
from config.config import config

# 获取服务器端口
port = config.server_port

# 获取Dify API密钥
api_key = config.dify_api_key

# 获取所有配置
all_config = config.get_all_config()
```

### 动态更新配置

```python
# 更新单个配置项
config.update_config("server", "port", 8080)

# 重新加载配置
config.reload()
```

## 注意事项

1. **安全性**: 不要在代码中硬编码敏感信息（如API密钥），使用配置文件或环境变量
2. **权限**: 确保配置文件有适当的读写权限
3. **备份**: 定期备份重要配置
4. **验证**: 启动前验证配置是否正确

## 故障排除

### 常见问题

1. **配置文件不存在**: 系统会自动创建默认配置文件
2. **配置文件格式错误**: 检查JSON格式是否正确
3. **权限不足**: 确保应用有权限读取配置文件
4. **环境变量未生效**: 重启终端或重新设置环境变量

### 调试方法

```python
# 打印当前配置
print(config.get_all_config())

# 检查环境变量是否生效
import os
print("DIFY_API_KEY:", os.getenv('DIFY_API_KEY'))
```
